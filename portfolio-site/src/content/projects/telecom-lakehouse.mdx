## Problem

Telecom billing teams sit on enormous CDR (Call Detail Record) volumes and
need a single source of truth for revenue, ARPU, and churn signals. The
Medallion pattern (Bronze raw, Silver normalized, Gold dimensional) keeps
late-arriving data and corrections tractable while exposing clean marts to
BI. This project is an end-to-end implementation: synthetic CDR generation,
Airflow orchestration, Iceberg storage on S3-compatible object storage, data
contracts via Great Expectations, and dbt for the analytics layer - all
runnable locally on docker-compose with no AWS account.

## Stack

- **Apache Airflow (LocalExecutor)** orchestrates ingest, transform, and dbt runs with retries and SLAs.
- **Apache Iceberg** chosen for partition evolution, hidden partitioning, and engine portability (Spark, Trino, Athena).
- **MinIO** locally, S3 in production - identical API.
- **Great Expectations** for column-level data contracts at Bronze (the layer where things break).
- **dbt** for Silver-to-Gold transforms with `unique`, `not_null`, `accepted_values`, and `relationships` tests.
- **Terraform** to provision the AWS variant (S3 + Glue + Athena) when you want to deploy for real.

## Data quality strategy

Bronze: Great Expectations suite enforces column types, nulls, and ranges before
data enters the lake.

Silver: dbt tests on every model. Failed tests block downstream Gold builds.

Gold: snapshot-style metrics with row count and freshness alerts.
