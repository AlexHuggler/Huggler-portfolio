## Problem

There is a real gap between "junior writes a working query" and "senior knows
when to broadcast, partition-prune, flatten a CTE, or rewrite a window
function." This tool closes part of that gap by sending parsed SQL plus
heuristic findings to Claude with a structured optimizer prompt, then
optionally verifying the rewrite against a real EXPLAIN plan. It targets
Spark SQL and Snowflake, the two dialects I work with most.

## Methodology

Fifty queries across five categories: join optimization, aggregation rewrites,
CTE flattening, partition pruning, broadcast-join opportunities. Each is
labeled with expected suggestion keywords and expected cost direction in
`corpus/ground_truth.yaml`. Suggestions are scored on three axes: keyword
overlap with ground truth, EXPLAIN cost delta (when the engine is available),
and human review.

## Honest limitations

- Claude can hallucinate column or table names that look plausible but do not exist in the schema.
- EXPLAIN-based scoring is only as good as the optimizer's cost estimates - a "cheaper" plan can still be slower in practice.
- Dialect drift between Spark SQL and Snowflake catches edge cases (e.g. `QUALIFY`, lateral views).
- Prompt sensitivity: small changes to the prompt template materially change suggestion quality.
- The CLI runs against `ANTHROPIC_API_KEY`; without it, only heuristic dry-run mode is available.

See the repo for the full corpus, prompt template, and CLI usage.
